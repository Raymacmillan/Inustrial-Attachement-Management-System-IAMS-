Industrial Attachment Management System (IAMS)

Project Overview
The Industrial Attachment Management System (IAMS) is a specialized digital solution developed to automate the manual industrial attachment processes for the Department of Computer Science. The system streamlines coordination between students, departmental coordinators, and host organizations.

Key Objectives:

Automated Registration: Online portals for both students and host organizations to submit their details and preferences.

Intelligent Matching: Implementation of automated heuristics to pair students with organizations.

Progress Tracking: Digital submission of weekly logbooks and final reports.

Centralized Assessment: Online capture of evaluation results from university and industrial supervisors.


Tech Stack
Frontend: React (Vite)

Backend/Database: Supabase (PostgreSQL)

Authentication: Supabase Auth

Hosting: Vercel

Workflow & Branching Strategy
This project adheres to the Scrum method, ensuring iterative development through defined releases.

Branching Strategy: We follow a feature-branch workflow. New features are developed in branches labeled feature/T-ID-description (e.g., feature/T-1.1-database-setup).


Merge Policy: All code must be merged into the main branch via Pull Requests.


Definition of Done (DoD): No feature is merged unless it is completed, reviewed, and successfully tested.
